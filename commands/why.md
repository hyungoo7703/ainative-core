description: 코드가 왜 이렇게 작성되었는지 분석
---

선택한 코드 또는 지정한 파일/함수가 왜 이렇게 작성되었는지 분석합니다.
이전 대화에서 대상이 이미 언급되었다면 다시 묻지 않고 바로 분석합니다.

## 분석 절차

1. **git blame / git log** — 누가, 언제, 어떤 커밋에서 이 코드를 작성/변경했는지
2. **커밋 메시지 확인** — 변경 이유가 기록되어 있는지
3. **관련 코드 추적** — 이 코드를 호출하는 곳, 의존하는 곳 확인
4. **대안 비교** — 다른 방식으로 작성할 수 있었는지, 왜 이 방식을 택했는지 추론

## 출력 형식

```
## 이 코드가 하는 일
(한 줄 요약)

## 왜 이렇게 작성되었는가
(git 히스토리, 커밋 메시지, 코드 맥락 기반 분석)

## 문제가 있는가
(있으면 지적, 없으면 "현재 구현이 적절합니다")

## 더 나은 방법이 있는가
(있으면 제안, 없으면 생략)
```

## 규칙

- 추측이 아닌 근거(git history, 코드 맥락)를 기반으로 설명
- "원래 이렇게 하는 겁니다" 같은 답변 금지 — 구체적 이유를 제시
- 사용자가 납득할 때까지 추가 질문에 응한다
