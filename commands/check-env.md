프로젝트 코드를 스캔하여 환경 관련 문제를 찾아냅니다.

## 검사 항목

### 1. 하드코딩된 값
- URL (localhost, 특정 IP, 특정 도메인)
- 포트 번호
- API 키, 토큰, 비밀번호
- 파일 경로 (절대 경로)
- 이메일, 전화번호 등 개인정보

### 2. 환경별 분기 누락
- 개발/스테이징/운영 환경에 따라 달라져야 하는 설정이 분기되지 않은 곳
- 환경 변수로 빠져야 하는데 코드에 직접 박혀있는 값
- .env.example에는 있는데 코드에서 참조하지 않는 변수 (또는 그 반대)

### 3. 임시 코드 방치
- TODO, FIXME, HACK, XXX 주석
- console.log, print, debugger 등 디버깅 코드
- 주석 처리된 코드 블록
- 임시 테스트 데이터

### 4. 설정 파일 정합성
- .env.example과 실제 .env 항목 비교
- 환경별 설정 파일이 있다면 (dev/staging/prod) 누락 항목 비교
- Docker/배포 설정의 환경 변수 일치 여부

## 출력 형식

- 🔴 **즉시 수정**: 보안 위험 또는 운영 환경에서 장애 유발 가능
- 🟡 **확인 필요**: 의도한 건지 확인이 필요한 항목
- 🟢 **정상**: 환경 설정이 잘 되어있는 부분

파일 경로와 라인 번호를 함께 표시합니다.
