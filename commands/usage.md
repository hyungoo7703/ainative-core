ainative-core에서 제공하는 커맨드와 에이전트 목록을 안내해주세요.

## 추천 워크플로우

프로젝트 전체 사이클에 맞춰 순서대로 사용하면 효과적입니다:

```
/research  → 레퍼런스 조사, 라이브러리 비교
/spec      → 요구사항 분석, 기능 명세
/init      → 기술 스택 선택 + 초기 세팅
/plan      → 구현 계획 수립
코딩 시작
/tdd       → 테스트 주도 개발 (필요 시)
/debug     → 버그 발생 시 체계적 디버깅
/review    → 코드 리뷰
/verify    → 완료 전 검증
/docs      → 산출물 생성
```

> `/tdd`, `/debug`는 매번 쓸 필요 없이, 필요한 상황에서만 사용합니다.
> - `/tdd`: 핵심 비즈니스 로직(결제, 인증, 데이터 처리 등) 개발 시
> - `/debug`: 원인을 모르는 버그가 발생했을 때

## 슬래시 커맨드

| 커맨드 | 설명 |
|--------|------|
| `/research` | 레퍼런스 및 라이브러리 비교 분석 |
| `/spec` | 요구사항 분석 및 기능 명세 작성 |
| `/init` | 새 프로젝트 기술 스택 선택 및 초기 세팅 |
| `/plan` | 작업 구현 계획 수립 |
| `/tdd` | 테스트 주도 개발 (Red-Green-Refactor) |
| `/debug` | 체계적 디버깅 (4단계 근본 원인 분석) |
| `/review` | 변경된 코드 리뷰 |
| `/verify` | 작업 완료 전 검증 체크리스트 |
| `/check-env` | 환경별 설정 누락 및 하드코딩 검증 |
| `/security` | 보안 취약점 검증 |
| `/continue` | 이전 계획에서 남은 작업 이어서 진행 |
| `/summarize` | 현재 작업 내용 요약 |
| `/docs` | 프로젝트 산출물 생성 (Mermaid 다이어그램 포함) |
| `/how-to-run` | 현재 프로젝트 실행 방법 안내 |
| `/usage` | 이 안내 표시 |

## 에이전트

| 이름 | 설명 |
|------|------|
| `reviewer` | 코드 리뷰 전문 에이전트 (깊이 있는 분석) |
| `planner` | 구현 설계 및 플래닝 전문 에이전트 |

## 적용 중인 규칙

| 규칙 | 내용 |
|------|------|
| `language` | 응답은 한국어, 코드/커밋은 영어 |
| `git-convention` | Conventional Commits 형식 |
| `coding-style` | 간결한 코드, 불필요한 추상화 금지 |
| `security` | 민감 정보 커밋 금지, 입력 검증 |

위 내용을 표 형태로 보여주세요.
