ainative-core에서 제공하는 커맨드, 스킬, 에이전트 목록을 안내해주세요.

## 추천 워크플로우

프로젝트 전체 사이클에 맞춰 순서대로 사용하면 효과적입니다:

```
/research  → 레퍼런스 조사, 라이브러리 비교
/spec      → 요구사항 분석, 기능 명세
/init      → 기술 스택 선택 + 초기 세팅
/plan      → 구현 계획 수립
코딩 시작
  (자동) review    → 코드 변경 시 자동 리뷰
  (자동) debug     → 에러 발생 시 자동 디버깅
  (자동) verify    → 작업 완료 시 자동 검증
/tdd       → 테스트 주도 개발 (필요 시)
/check-env → 환경 설정 검증
/security  → 보안 취약점 검증
/docs      → 산출물 생성
```

> `/tdd`는 핵심 비즈니스 로직(결제, 인증, 데이터 처리 등) 개발 시 사용합니다.

## 슬래시 커맨드

수동으로 실행하는 커맨드입니다.

| 커맨드 | 설명 |
|--------|------|
| `/research` | 레퍼런스 및 라이브러리 비교 분석 |
| `/spec` | 요구사항 분석 및 기능 명세 작성 |
| `/init` | 새 프로젝트 기술 스택 선택 및 초기 세팅 |
| `/plan` | 작업 구현 계획 수립 |
| `/tdd` | 테스트 주도 개발 (Red-Green-Refactor) |
| `/check-env` | 환경별 설정 누락 및 하드코딩 검증 |
| `/security` | 보안 취약점 검증 (OWASP 기반) |
| `/summarize` | 현재 작업 내용 요약 |
| `/docs` | 프로젝트 산출물 생성 (Mermaid 다이어그램 포함) |
| `/how-to-run` | 현재 프로젝트 실행 방법 안내 |
| `/continue` | 이전 계획에서 남은 작업 이어서 진행 |
| `/usage` | 이 안내 표시 |

## 스킬 (자동 활성화)

상황에 맞게 자동으로 활성화됩니다. 수동으로 실행하려면 `/스킬이름`으로 호출하세요.
예: `/review`, `/debug`, `/verify`

| 스킬 | 설명 | 활성화 시점 |
|------|------|-------------|
| `review` | 코드 리뷰 | 코드 변경, git diff, PR 작성 시 |
| `debug` | 체계적 디버깅 (4단계) | 에러 발생, 버그 수정 시 |
| `verify` | 작업 완료 전 검증 | 작업 완료 시점 |
| `refactor` | 단계적 리팩토링 | 코드 구조 변경 시 |
| `api-design` | API 설계 가이드 | API 엔드포인트 설계 시 |
| `error-handling` | 에러 처리 패턴 | try-catch, 에러 핸들링 시 |
| `performance` | 성능 최적화 체크 | 성능 이슈, 최적화 요청 시 |
| `accessibility` | 웹 접근성 검사 | HTML, UI 컴포넌트 작성 시 |
| `pr-description` | PR 작성 가이드 | PR 생성 시 |
| `code-review-response` | 리뷰 피드백 반영 | 리뷰 코멘트 처리 시 |
| `explain` | 코드/개념 설명 | 설명 요청, "이게 뭐야" 질문 시 |

## 에이전트

복잡한 작업을 격리된 컨텍스트에서 처리합니다.

| 이름 | 설명 |
|------|------|
| `reviewer` | 코드 리뷰 전문 에이전트 (깊이 있는 분석) |
| `planner` | 구현 설계 및 플래닝 전문 에이전트 |

## 적용 중인 규칙

항상 자동으로 적용됩니다.

| 규칙 | 내용 |
|------|------|
| `language` | 응답은 한국어, 코드/커밋은 영어 |
| `git-convention` | Conventional Commits 형식 |
| `coding-style` | 간결한 코드, 불필요한 추상화 금지 |
| `security` | 민감 정보 커밋 금지, 입력 검증 |

위 내용을 표 형태로 보여주세요.
