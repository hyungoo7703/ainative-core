---
description: 작업 구현 계획 수립
---

주어진 작업에 대해 구현 계획을 세워주세요.
이전 대화에서 이미 작업이 결정되었다면 다시 묻지 말고 바로 계획을 수립합니다.
**프로젝트에 spec 문서(CLAUDE.md, docs/spec.md 등)가 있으면 반드시 읽고 참조합니다.**

1. **명세 확인**: spec 문서가 있으면 해당 기능의 명세를 확인
2. **목표 정리**: 무엇을 달성하려는지 한 줄 요약
3. **현재 상태 파악**: 관련 코드/파일 확인
4. **구현 단계**: 순서대로 나열 (각 단계는 하나의 커밋 단위)
5. **리스크**: 주의할 점, 영향 범위
6. **자가 검증**: 이 계획에 보안, 성능, 확장성 문제가 없는지 스스로 검토한다. 문제가 있으면 계획을 수정한 뒤 사용자에게 제시한다.
7. **확인 방법**: 완료 후 어떻게 검증할지 (spec 대비 체크)

계획이 합의되면 바로 구현을 시작합니다.
