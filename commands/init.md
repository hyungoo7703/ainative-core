새 프로젝트를 시작할 때 기술 선택부터 초기 세팅까지 가이드합니다.

## 순서

### 1. 목적 파악
사용자에게 질문:
- 뭘 만들 건지? (웹앱, API 서버, CLI 도구, 모바일앱, 확장 프로그램, MCP 서버 등)
- 누가 쓸 건지? (본인만, 팀, 외부 사용자)
- 규모는? (프로토타입, 사이드 프로젝트, 프로덕션)

### 2. 기술 스택 추천
목적에 맞는 선택지를 비교하여 제안:

**언어/프레임워크**
- 왜 이걸 추천하는지 이유와 함께 제시
- 최소 2개 선택지 비교

**아키텍처**
- 풀스택 통합 vs 프론트/백 분리
- 모놀리스 vs 마이크로서비스

**데이터베이스** (필요 시)
- RDB vs NoSQL vs 파일 기반
- ORM 사용 여부

**배포** (필요 시)
- 로컬 전용 vs 클라우드
- 플랫폼 추천 (Vercel, Railway, Docker 등)

### 3. 사용자 확인
추천 스택을 정리해서 보여주고 확인을 받는다. 사용자가 동의하면 다음 단계로.

### 4. 프로젝트 초기 세팅
- 디렉토리 구조 생성
- 패키지 매니저 초기화
- 기본 설정 파일 생성 (.gitignore, tsconfig, eslint 등)
- 의존성 설치
- 개발 서버 실행 확인

### 5. 다음 단계 안내
- `/plan`으로 구현 계획 수립 권장
- 주요 참고 문서 링크 제공

## 규칙

- 사용자가 이미 정한 것이 있으면 존중한다
- 과도한 boilerplate를 넣지 않는다
- 선택이 애매하면 가장 단순한 옵션을 기본값으로 추천한다
