description: 레퍼런스 및 라이브러리 비교 분석
---

주어진 주제에 대해 레퍼런스를 조사하고 비교 분석합니다.

## 절차

### 1. 조사 대상 확인
- 사용자에게 무엇을 비교하고 싶은지 확인
- 예: 프레임워크, 라이브러리, 아키텍처 패턴, 서비스 등

### 2. 후보 선정
- 주요 선택지 2~4개를 선정
- 각각의 공식 문서, GitHub, npm/pypi 등에서 정보 수집

### 3. 비교 분석
다음 기준으로 비교표를 작성:

| 기준 | 설명 |
|------|------|
| 기능 적합성 | 요구사항을 얼마나 충족하는가 |
| 성숙도 | 릴리즈 이력, 커뮤니티 크기, GitHub 스타 |
| 유지보수 | 최근 업데이트, 이슈 응답 속도 |
| 학습 곡선 | 도입 난이도 |
| 생태계 | 플러그인, 통합, 문서 품질 |
| 성능 | 벤치마크 (해당 시) |

### 4. 추천
- 비교 결과를 바탕으로 추천안 제시
- 추천 이유를 명확히 설명
- 트레이드오프도 함께 언급

## 규칙

- 최신 정보 기준으로 비교 (deprecated된 것은 제외)
- 주관적 의견이 아닌 근거 기반으로 비교
- 사용자의 프로젝트 맥락을 고려하여 추천
