요구사항을 정리하고 구체적인 기능 명세로 변환합니다.

## 절차

### 1. 요구사항 수집
이전 대화에서 이미 언급된 요구사항이 있으면 활용하고, 부족한 부분만 추가로 질문합니다.
- 누가 쓰는가? (사용자 유형)
- 핵심 기능은 무엇인가?
- 반드시 있어야 하는 것 vs 있으면 좋은 것
- 제약 조건 (기술, 환경, 일정 등)

### 2. 기능 분류

| 우선순위 | 설명 |
|----------|------|
| **P0 (필수)** | 없으면 출시 불가 |
| **P1 (중요)** | 첫 번째 릴리즈에 포함 권장 |
| **P2 (선택)** | 이후 반복에서 추가 |

### 3. 기능 명세 작성
각 기능에 대해:

```
## 기능명
- 설명: 한 줄 요약
- 사용자 스토리: "~로서, ~를 하고 싶다, ~하기 위해"
- 입력/출력: 무엇을 받고 무엇을 반환하는가
- 예외 상황: 에러 케이스
- UI 흐름: (해당 시) 화면 전환 설명
```

### 4. 범위 확인
- 정리된 명세를 사용자에게 보여주고 확인
- 빠진 것, 불필요한 것 조정
- MVP 범위 합의

### 5. 명세 저장
- 합의된 명세를 **CLAUDE.md** 또는 **docs/spec.md**에 저장한다
- 이후 `/plan`, `/verify` 등에서 참조할 수 있도록 프로젝트에 남긴다

## 규칙

- 사용자가 말한 것만 명세에 포함 (추측으로 기능 추가 금지)
- 기술 구현이 아닌 사용자 관점에서 작성
- 모호한 요구사항은 질문으로 명확히 한다
- 명세는 반드시 파일로 저장하여 이후 단계에서 참조 가능하게 한다
