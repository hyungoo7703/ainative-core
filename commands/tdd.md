description: 테스트 주도 개발 (Red-Green-Refactor)
---

TDD 사이클로 작업을 진행합니다. 반드시 테스트를 먼저 작성하세요.

## 사이클

1. **RED** — 실패하는 테스트 먼저 작성
2. **GREEN** — 테스트를 통과하는 최소한의 코드 구현
3. **REFACTOR** — 테스트 통과 상태를 유지하며 코드 개선

## 규칙

- 테스트 없이 코드를 작성하지 않는다
- 테스트는 행동(behavior)을 검증한다, 구현 세부사항이 아니라
- 한 번에 하나의 테스트만 추가한다
- 테스트를 수정해서 통과시키지 말고, 코드를 수정한다

## 필수 테스트 케이스

- 정상 동작 (happy path)
- 엣지 케이스 (null, 빈 값, 경계값)
- 에러 상황
- 외부 의존성은 mock 처리
