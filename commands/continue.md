이전에 세운 계획이나 진행 중이던 작업에서 남은 것을 확인하고 이어서 진행합니다.

## 절차

1. **현재 상태 파악**
   - git log, git diff, TODO 목록, 기존 대화 컨텍스트를 확인
   - 완료된 것과 남은 것을 구분

2. **남은 작업 정리**
   - 미완료 항목을 우선순위 순으로 나열
   - 각 항목의 예상 작업량 표시

3. **확인**
   - 사용자에게 남은 작업 목록을 보여주고 확인
   - 순서 변경이나 제외할 항목이 있는지 질문

4. **재개**
   - 확인된 순서대로 작업 시작
