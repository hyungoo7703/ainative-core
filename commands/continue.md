이전에 세운 계획이나 진행 중이던 작업에서 남은 것을 확인하고 이어서 진행합니다.
이전 대화에서 맥락이 있으면 활용하고, 이미 파악된 정보를 다시 묻지 않습니다.

## 절차

1. **현재 상태 파악**
   - git log, git diff, TODO 목록, 기존 대화 컨텍스트를 확인
   - 완료된 것과 남은 것을 구분

2. **남은 작업 정리**
   - 미완료 항목을 우선순위 순으로 나열
   - 각 항목의 예상 작업량 표시

3. **다음 작업 추천**
   - 우선순위 기준으로 다음 작업을 추천하고 바로 진행 의사를 묻는다
   - 불필요하게 전체 목록을 나열하며 선택을 강요하지 않는다

4. **재개**
   - 확인되면 바로 작업 시작
