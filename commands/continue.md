이전에 세운 계획이나 진행 중이던 작업에서 남은 것을 확인하고 이어서 진행합니다.
이전 대화에서 맥락이 있으면 활용하고, 이미 파악된 정보를 다시 묻지 않습니다.

## 절차

1. **git 상태 점검**
   - `git branch`, `git log --oneline -10`, `git status` 실행
   - 현재 브랜치가 이미 머지된 브랜치인지 확인 (main과 비교)
   - 머지 완료된 브랜치라면: main으로 전환 → pull → 새 브랜치 생성 안내
   - main이 remote보다 뒤처져 있으면 pull 먼저 실행
   - 머지 완료되어 불필요한 브랜치가 있으면 정리 (로컬: `git branch -d`, 원격: `git push origin --delete`)

2. **현재 상태 파악**
   - spec 문서(CLAUDE.md, docs/spec.md)가 있으면 읽고 참조
   - git log, git diff, TODO 목록, 기존 대화 컨텍스트를 확인
   - 완료된 것과 남은 것을 구분

3. **남은 작업 정리**
   - 미완료 항목을 우선순위 순으로 나열
   - 각 항목의 예상 작업량 표시

4. **다음 작업 추천**
   - 우선순위 기준으로 다음 작업을 추천하고 바로 진행 의사를 묻는다
   - 불필요하게 전체 목록을 나열하며 선택을 강요하지 않는다

5. **재개**
   - 확인되면 바로 작업 시작
